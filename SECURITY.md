# Security

If you find a security issue, please use GitHub's private
["Report a vulnerability"](../../security/advisories/new) feature instead of a
public issue, so it isn't disclosed before there's a fix.

For anything else - false-positive antivirus flags, SmartScreen questions, or
general "is this safe" questions - a regular [issue](../../issues/new) is fine;
see the README's [What it does with your files](README.md#what-it-does-with-your-files)
section first, it answers most of these directly.
